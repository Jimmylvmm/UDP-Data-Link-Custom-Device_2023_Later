UDP Custom Device Pages

Introduction

The UDP Custom Device pages are used in a Veristand system definition to configure UDP packets that will be transacted across a network. It is important to make sure that all the pages are setup correctly to ensure that data is exchanged correctly. Certain errors in configuration will further prevent the system definition from deploying.

Pages Listing

Below is a quick listing of all the basic pages available in the UDP Custom Device.

Page Description
UDP Custom Device Main Page The primary landing page when a new instance of the UDP Custom Device is created. Contains global import/export configuration and timestamp channel selection.
Transmit Packets Enumerates all the transmit packets currently setup. Provides menu buttons for creating and removing transmit packets.
Transmit Packet A singular transmit packet. Contains information regarding the NIC, port, and other packet-wide configuration.
Transmit Channel Configuration The configuration of the transmit packet. Allows for transmit packet/channel information to be imported from file.
Receive Packets Enumerates all the receive packets currently setup. Provides menu buttons for creating and removing receive packets.
Receive Packet A singular receive packet. Contains information regarding the NIC, port, and other packet-wide configuration.
Receive Channel Configuration The configuration of the receive packet. Allows for receive packet/channel information to be imported from file.
Receive Channel A single channel of data to be received over UDP. Contains information about the expected data type and location the data is contained in the receive packet.
Buffered Transmit Packet A transmit packet that operates on buffers of data to send on a UDP network connection. Powerful when transmitting large payloads to external applications directly from other custom devices or models.
Buffered Receive Packet A receive packet that populates buffers of data from data received over a UDP network connection. Powerful when receiving large payloads of external data that needs to passed into one more models or custom devices.
