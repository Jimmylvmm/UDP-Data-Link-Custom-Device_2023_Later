Receive Channel Configuration

Introduction

This page provides a means for importing a Channel Configuration File (*.csv) for data types, data order, and channels in the receive packet. Unlike the Transmit Channel Configuration page, applying this configuration will create all necessary receive channels if they do not exist already.

Configuration Summary

Channel data in a receive packet does not need to be manually reordered and have data types assigned using the configuration screen. Instead, a *.csv file can be used to import an existing definition for all data expected in the receive packet. Examples of thise file can be found with the custom device distribution. The configuration file import will modify any existing channels with matching names, and create any necessary channels to satisfy the remaining definition in the configuration file. Any extraneous channels will also be removed. The configuration file not be fully imported and applied until the Apply Config button is pressed in the top menu.

When the Apply Config button is pressed, the contents of the configuration file will take priority over any other channels, data typing, or order previously established for the receive packet. All the old channels will be removed and new channels will be added to best match the configuration file contents. Additional channels may be added after the configuration file is applied.

Description
Channel Configuration File The target configuration file (*.csv) to import for data type and data ordering. The configuration is loaded immediately and displayed. It is not applied until the Apply Config button is pressed in the top menu.
Imported Channel Configuration Provides a summary of the contents of the Channel Configuration File. This list cannot be manipulated manually. Dispalys the target channel names, the Index/Order that they should be sent, and the data type that they should be encoded with.
