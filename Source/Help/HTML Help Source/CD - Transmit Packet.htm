Transmit Packet

Introduction

This page represents a single packet to be transmitted out on a UDP port. A packet will have an ID, which represents the relative order in which the packet will be sent. This is the primary means for formatting and sending channel data out on a UDP port.

Configuration Summary

A transmit packet must be configured with a local NIC and port to send on. Additionally, the packet needs a destination address and port to try to send to. Further configuration can adjust the endianness of the packet contents, the packet header, and frequency the packet is transacted. A packet header must have at least the Packet Size bytes. Channels to send in the transmit packet may be added using the Add Channel(s) button or right-click menu option from the system definition tree.

Description
Configure Header Clicking on this button opens a dialogue that displays the current header payload that will be used for the transmit packet. Header elements may be removed and added as needed, but the Packet Size bytes are required at a minimum for the header.
Endianness Determines the byte endianness of the transmitted packet. Supports little-endian and big-endian.
IP Address of Local NIC IP address of the local network interface card to use to send this packet.
IP Address of Destination IP address of the destination that is going to receive this packet. When the Multicast? option is set active, this field is used for the Multicast address.
Port The local port to use to send this packet.
Destination Port IP address of the destination that is going to receive this packet.
Decimation Determines the rate at which this packet is sent. This rate is relative to the loop rate of the Primary Control Loop (PCL). If the PCL is set to 100 Hz, and the decimation of this packet is set to 2, this transmit packet will be sent at a rate of 50 Hz.
TTL Sets the Time-To-Live property for transmitted packets. This is necessary when traversing particular pieces of networking equipment.
Auto-Assign Local NIC A shortcut functionality that enables the auto-population of the IP Address of Local NIC field. Attempts to use the IP address assigned to controller or realtime host.
Port as Channel When active, creates a Port channel. This port channel can be modified at runtime to change the destination port a packet is being sent to.
Debug Mode When active, logs verbose error messages to the Veristand console and default Veristand log file. Only errors are logged. Issues with opening duplex connections are current unhandled. Additionally, generates unique log files for the enabled packet. These log files can be found in the custom device directory on the target in the 'UDP_CD_Debug_Logs' directory. The custom device directory varies by target. The log file creation will be displayed at startup on the console. These log files are unmanaged and should be manually cleaned-up or removed from the target upon completion of debug. Debug logs automatically begin overwriting old records after the file size has exceeded ~24MB. This caps the log file size around 24MB.
NOTE: Debug mode sacrifices significant performance for thorough logging of errors. This includes receive packet timeouts and other potential error conditions.
Transmit Options This parameter controls the behavior of the Transmit channel. All transmit behavior is pending valid buffers, transmit bytes, and ports as relevant.
Always: This packet will always attempt to be sent, regardless of the value of the Transmit channel.
One-Shot: This packet will be sent any time the Transmit channel changes to a non-zero value. For example, this packet is transmitted when a change from 1 to 2 is observed, but not transmitted when a change from 1 to 0 is observed.
Toggle: This packet is sent as long as the value of the Transmit channel is non-zero.
Transmit Echo Options Determines the behavior of the Transmitted channel. These conditions only occur upon successful transmission of a packet.
PCL Reset: The Transmitted channel will set to 1 for approximately least one PCL cycle before returning to 0.
Counter: The Transmitted channel will continuously count up every time a packet is sent.
Mirror: The Transmitted channel will display the same value as the Transmit channel.
Tx Channels Enumerates ALL THE CHANNELS configured to be transmitted with this transmit packet, as well as their expected data type, order in the packet, and the path to their source channel. A channel may be removed by selecting it from this list, right-clicking, and selecting Remove Channels. The data type of a channel may be changed similarly. Channel transmit order may be changed by dragging and dropping channels in this listbox.
Export Exports the current transmit packet channel configuration to a formatted csv file. This csv file may be imported by transmit or receive channel configurations for quick configuration and setup of many channels.

Channels

Description
Transmit When this channel is set to 1, the channel data is transmitted out on the network. This channel resets back to 0 after transmitting a payload. This channel is only used when the Always Transmit property is inactive.
Transmitted Toggles high for one PCL cycle to confirm that successfully transmitted the packet. Depending on the Transmit Options property, the behavior of this channel may vary. For example, this channel will always have a value of 1 when the Always option is used.
Port Only present when the Port as Channel property is active. Initializes to a default value that is the same as the Destination Port property. When modified at runtime, results in packets being delivered to a different port. Prevents outbound traffic when set to 0.
