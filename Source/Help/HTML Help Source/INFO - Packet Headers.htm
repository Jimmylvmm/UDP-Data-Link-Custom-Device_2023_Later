Packet Header

Introduction

The packet header is used for communicating information about packet payloads to the packet receivers. These always preceed the actual data payloads in the transacted packet. The packet header is customizable (with some limitation). Packet headers are only relevant when working with non-buffered packets. It is advised when working within the UDP Custom Device that the transmitting and receiving endpoints of a packet have the same header. Failure to use the same header will result in parsing errors.

Packet Headers

Required? Size (bytes) Description
Packet Size Yes 2 Required in all packets. Occupies the first two bytes of a packet to communicate the size of the packet to the receiver. Includes the header and the channel data.
Protocol Revision No 1 Tracks the revision of the of the Veristand Data Library and payload structuring for a packet. Should be statically set to 1.
System Definition Version No 8 Tracks the version of the system definition. The two most significant bytes are staticaly set to zero. Starting at the third most significant byte, the payload breakdown looks like this:
[Major] - 1 Byte
[Minor] - 1 Byte
[Fix] - 2 Bytes
[Build] - 2 Bytes
Source Node Name No 32 A byte array that contains the first 32 bytes of the name of the Veristand Controller that produced the transmitted packet.
Packet ID No 4 Transmit: Identifies the index/order of the packet, as set in the system definition file.
Receive: Identifies the index of the transmit packet this receive packet is expecting to handle.
Note: Multi-packet transmission is not current supported on the same port.
Channel Count No 4 Identifies the number of post-header channel data fields are present in the packet. Does not indicate the number of bytes the channels are using.
Sequence Number No 8 Identifies the current transmit count of the packet. Starts at 0 and increments each time a packet is sent.
Timestamp No 8 Transmit:The current value on the Timestamp channel.
Receive:Not actively used at this time by receive packets. This field still needs to be allocated in the header if it is present in a transmission payload.
