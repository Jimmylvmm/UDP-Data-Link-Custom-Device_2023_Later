Receive Channel

Introduction

A specialized channel that will get data from a received UDP packet.

Configuration Summary

Receive channels are slightly specialized in that they track the data type that they represent in the received packet, as well as their location in the packet (relative to the front of the packet). Receive channels start with a default value of 0.

Description
Field Index The relative location of the corresponding packet data that this channel will hold. An index of 0 indicates that this channel will get the first-most data from the received packet (after the header). An index of 1 means that this channel will get the second-most data from the received packet.
Data Type The expected data type that this channel will receive from the inbound UDP packet. Boolean and Single are the only two data types currently supported.
